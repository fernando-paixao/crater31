Configuração de acesso RTSP

Abaixo estão as etapas para configurar o acesso RTSP a um DVR:

Resumo:

Essas etapas são necessárias para configurar o acesso ao RTSP a um DVR porque estabelecem a infraestrutura de rede necessária para permitir a comunicação entre a rede externa (Internet) e o DVR interno.

  1. O domínio fornece o endereço para acessar o DVR remotamente.
  2. encaminhamento de porta direciona solicitações RTSP da Internet para o DVR dentro da rede local.
  3. Atribuir um fixo IP ao DVR garante que seu endereço de rede permaneça constante para comunicação consistente.

Detalhes da etapa:

  1. Domínio (DVR ou roteador):
    O domínio é o endereço através do qual você acessa o DVR (ou roteador) remotamente.
    Normalmente é um nome de domínio ou endereço IP.
    Esta etapa é crucial porque identifica onde o DVR pode ser alcançado na Internet.

    DDNS DVR Intelbras No-IP no modem Vivo
  2. Port Forward (roteador):
    O encaminhamento da porta é uma técnica usada pelos roteadores para permitir que dispositivos externos acessem os serviços fornecidos pelos dispositivos na rede interna.
    Como os DVRs geralmente se comunicam sobre portas específicas, como a porta RTSP (normalmente a porta 554), o encaminhamento de portas é necessário para direcionar solicitações externas de RTSP para o DVR dentro da rede local.
    Sem encaminhamento de portas, pedidos externos seriam bloqueados pelo firewall do roteador.

    Liberar portas modem da Vivo
  3. IP Fixo (DVR):
    Um IP fixo (também conhecido como IP estático) é um endereço IP atribuído a um dispositivo que permanece constante, diferentemente de um endereço IP dinâmico que pode mudar com o tempo.
    A atribuição de um IP fixo ao DVR garante que seu endereço de rede permaneça consistente.
    Isso é importante porque, ao configurar o encaminhamento da porta, o roteador precisa conhecer o endereço IP interno do DVR para encaminhar solicitações RTSP para ele.
    Se o endereço IP do DVR mudar (o que pode acontecer com a atribuição dinâmica de IP), a regra de encaminhamento de porta se tornaria inválida e o acesso remoto ao DVR seria interrompido.

    RSTP Playback
        Para testar um link RTSP como o que você forneceu, você pode usar vários softwares de mídia ou ferramentas de linha de comando que suportam o streaming RTSP.        
        
        ffplay
            Uma ferramenta popular de linha de comando para esse fim é o FFPLAY, que faz parte da estrutura multimídia do FFMPEG.
            Veja como você pode usar o FFPLAY para testar o link RTSP:
            
            Na linha de comando
                ffplay rtsp://domain:port/cam/realmonitor?channel=1&subtype=0
        VLC
            A popular tool to open video and streams.

            Na interface do usuário do programa
                Abra VLC Media Player.
                Vá em abrir Mídia de Fluxo de Rede.
                Coloque seu link RTSP (rtsp://domain:port/cam/realmonitor?channel=1&subtype=0) no campo URL da rede.
                Se estiver tudo certo em alguns instantes as imagens da câmera serão mostradas na tela

            Na linha de comando
                vlc rtsp://domain:port/cam/realmonitor?channel=1&subtype=0
